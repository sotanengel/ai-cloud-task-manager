# CloudPilot エラーコード一覧

AIエージェントはこのドキュメントを参照して、エラーの原因と修正方法を確認できます。

## L1: 構文検証エラー

| コード | 説明 | 修正方法 |
|---|---|---|
| `E1001` | YAML/JSONの構文が不正 | 有効なYAMLまたはJSONであることを確認する |

## L2: スキーマ検証エラー

| コード | 説明 | 修正方法 |
|---|---|---|
| `E1002` | 必須フィールドが欠落 | エラーの `field` に示されたフィールドを追加する |
| `E1003` | 列挙値が不正 | `suggestion` に示された許可値のいずれかを使用する |
| `E1004` | 未定義フィールドが含まれている | スキーマで定義されていないフィールドを削除する |
| `E1005` | 型が不一致 | フィールドの型をスキーマの定義に合わせる |
| `E1006` | フォーマットが不正 | 正規表現パターンやフォーマット要件を確認する |
| `E1007` | 値が許可範囲外 | 値を許可された範囲内に収める（例: replicas: 1〜20） |

## L3: 意味検証エラー

| コード | 説明 | 修正方法 |
|---|---|---|
| `E2001` | metadata.name の命名規則違反 | 小文字英数字とハイフンのみ、3〜63文字 |
| `E2002` | 参照先リソースが存在しない | 参照しているリソースが正しく定義されているか確認する |
| `E2003` | production環境で承認が未設定 | `spec.approvalRequired: true` を設定する |

## L4: ポリシー検証エラー

| コード | 説明 | 修正方法 |
|---|---|---|
| `E3001` | 許可されていないプロバイダ | `aws`, `gcp`, `azure` のいずれかを使用する |
| `E3002` | 許可されていないリソース種別 | `spec.constraints.allowedResourceTypes` の値を確認する |
| `E3003` | コスト上限超過 | リソース数やサイズを削減してコストを抑える |
| `E3004` | セキュリティポリシー違反 | Shisho Guard/trivy のスキャン結果を確認する |
| `E3005` | 破壊的操作に承認が必要 | `spec.approvalRequired: true` を設定し、人間の承認を得る |

## L5: ドライランエラー

| コード | 説明 | 修正方法 |
|---|---|---|
| `E4001` | ドライラン適用失敗 | 差分の内容を確認し、適用可能な状態かを確認する |
| `E4002` | 権限不足 | クラウドのIAMロール・権限設定を確認する |

## エラー応答の形式

```json
{
  "status": "rejected",
  "phase": "schema_validation",
  "taskId": null,
  "errors": [
    {
      "code": "E1003",
      "field": "spec.provider",
      "severity": "error",
      "message": "spec.provider must be one of [aws, gcp, azure]",
      "received": "aws-east",
      "suggestion": "spec.provider に 'aws' を指定してください。"
    }
  ]
}
```
